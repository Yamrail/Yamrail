目的
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

YamRail の文書体系は以下の3層で構成される。

  Core Lite（v0.7.0）:  軽量実行仕様。毎回プロンプトに読み込む。
  Core Full（v0.6.7）:  完全実行仕様。詳細定義の正典。参照用。
  思想書（本書・Guidebook）: 背景説明。人間向け。更新可能。

本ルールは、これらを混在させず、
矛盾なく役割を保ったまま共存させるための規則を定義する。

| 文書 | 読み手 | 用途 | 更新頻度 | 依存関係 |
|------|--------|------|----------|---------|
| Core Lite v0.7.0 | LLM | 軽量実行仕様 | 低（安定） | Core Full を参照 |
| Core Full v0.6.7 | LLM | 完全実行仕様 | 中 | Core Lite を参照しない |
| 思想書（本書） | 人間 | 背景説明 | 高 | Core を参照可 |


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rule_1 — 依存関係は一方向である
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（旧 Rule_1 + Rule_8 を統合）

依存関係の方向:
  思想書 → Core を参照してよい
  Core Lite → Core Full を参照してよい
  Core Full → Core Lite を参照しない
  Core Full → 思想書を参照しない

理由:
  Core Full は毎回読み込む完全仕様であり、
  軽量化と安定性を保つために外部文書への依存を持たない。
  Core Lite は詳細定義を Core Full に委譲するため参照を許可する。

読者（人間）は思想書・Core Lite・Core Full の
すべてを参照することで YamRail を完全に理解できる。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rule_2 — 思想書は Core を説明するが、Core を変更しない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
（旧 Rule_2 + Rule_7 を統合）

思想書は Core の背景・意図・思想を説明する文書である。
思想書の内容は Core の仕様・挙動を上書きしない。
Core の最終的な挙動はすべて Core 仕様書（v0.7.0 / v0.6.7）が決定する。

  思想書の役割: 理解のための背景説明
  Core の役割:  実行のための仕様定義


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rule_3 — 用語の意味は思想書で説明されるが、挙動は Core が決める
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

MeaningKernel や ParadoxRail などの
概念的な意味付けは思想書（Appendix A）で行う。

ただし、以下の挙動定義は Core 仕様書が決定する:
  CoreSafety.v1 が決定する挙動:
    - どの入力を danger_zone として force_terminate するか
    - どの入力を conceptual_zone として通常処理するか
  CalibrationLayer が決定する挙動:
    - どのトリガーで full_calibration / light_calibration が発動するか
  paradox_engine が決定する挙動:
    - パラドックスをどの分類（dissolution / transcendence / containment / weaponization）に分類するか
    - weaponization の安全リミッター閾値（max_weaponization_temperature）

  対応仕様: Core Lite v0.7.0 / Core Full v0.6.7


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rule_4 — Core の更新は思想書に反映されるが、思想書の更新は Core に影響しない
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Core のバージョン更新があれば、
思想書の巻末 Appendix C（更新履歴）に変更点を追加する。

更新責務:
  Core の更新を思想書に反映する責務は、
  当該バージョンの設計者（本仕様書では「単一の人間設計者」）が担う。
  自動更新は行わない（Core Lite v0.7.0 > minimal_declaration 参照）。

逆に、思想書の文章追加・修正は Core の挙動に影響を与えない。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rule_5 — Core はモデル向け、思想書は人間向け
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Core（Lite / Full 共通）の性質:
  - 構造定義のみ
  - 安定・更新頻度が低い
  - 比喩を使わない（挙動定義のため）
  - LLM が読み込む実行仕様

思想書の性質:
  - 背景説明・意図の解説
  - 比喩や例示を含んでよい
  - 更新可能
  - 人間が読む理解補助文書

両者は用途が異なるため、明確に分離する。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Rule_6 — 用語の翻訳は思想書の対照表で行う
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Core に登場する用語は、
思想書巻末の Appendix A で自然言語に対応付けられる。

対照表は以下の双方向の理解を補助する:
  Core 用語 ⇔ YamRail 用語 ⇔ 一般日本語

対応仕様: Guidebook v0.7.1 > Appendix A

まとめ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

本ルールは以下の3点を保証する。

  役割分離:
    Core Lite（軽量実行）・Core Full（完全仕様）・思想書（背景説明）の
    役割を明確に分離する。

  矛盾回避:
    思想書の更新が Core の挙動に影響しない構造を保つ。

  一方向依存:
    依存関係を「思想書 → Core Lite → Core Full」の一方向に限定し、
    Core Full が外部文書を参照しないことで安定性を確保する。
